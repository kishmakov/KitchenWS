Kitchen Computational Platform

What is Kitchen?


Kitchen is a computational platform designed for convenient modularization of computational algorithms and mathematical models. It designed with focus on convenience of reusability of published result.

Kitchen Structure

Idea


First and most important concept of the Kitchen is the computational element. Computational element is a building block and the product of the system simultaneously. From the philosophically point of view, computational element is a "potentiality" which turns into a "certainty" once "definiteness" is provided. See Computational Element section for more details.

Next concept of the Kitchen is the target function. Target function is the functional which maps computational element into number tuple. This mapping means evaluation of the element.

We distinguish two types of target functions:

  • "predicates" — functionals with discrete range. Those are used for "pass/fail" setting.
  • "scorings" — functionals which range is equipped with some partial order. Those are used for incremental improvement setting.

There are number of aspects which were addressed during Kitchen design. These are:

  • formulation of work target via organically supported target functions
  • test-oriented assembly mechanism embedded into platform tools
  • organic support of multilingual documentation

Implementation

Kitchen consists of

  • KitchenScript language
  • documentation system
  • assembly control system

KitchenScript is statically-typed programming language which could be compiled into a number of platforms, including JavaScript. When compiled to JavaScript, KitchenScript modules turned into a building blocks for computational algorithm assembly.

During the assembly process components are compulsory verified for type correspondence and optionally pre-tested before joining together. Pre-testing is aimed to diminish probability of erroneous misusage of the component functionality.

Once assembly process is finished, resulted modularized algorithm would possess embedded documentation. This documentation structure is a reflection of modular structure of the algorithm.
