Kitchen Types Representation

Primitive Types and Lists


There are number of primitive types in Kitchen Script. We use this types to initiate the process of type construction. Those are:


  • Boolean
  • Integer
  • Double

With every type T we have an list of this type denoted as [T].


Plain Objects


Plain objects are constructed as JSON objects with type value equal PO. Object member names are all other keys of the describing JSON objects. Corresponding values are type description of the members.


For instance, object for geographic point could be represented as 


{
  ...
  "GeographicPoint": {
    "type" : "PO",
    "longitude": "Double",
    "latitude": "Double"
  },
  ...
}

We could use already defined types as well. 


{
  ...
  "Country": {
    "type": "PO",
    "area": "Double",
    "population": "Integer",
    "cities": ["GeographicPoint"],
    "christiansDiscriminated": "Boolean"
  },
  ...
}

Functions


Functions are represented as JSON objects with type value equal F.


For instance, if we want to describe a type for function which returns a list of cities of given country which are within specified distance from specified point, it could be represented as 


{
  ...
  "NeighbourhoodCities": {
    "type": "F",
    "result": ["GeographicPoint"],
    "0": "Country",
    "1": "Double",
    "2": "GeographicPoint"
  },
  ...
}
